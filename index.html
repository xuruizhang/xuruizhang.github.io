Java语言概述

主要是考察一些基本的常识记忆性的知识。主要可包括以下几点：

  1. Java语言特性

  2. Java核心机制

  3. JVM JRE JDK 三者的区别联系

  4. Java的编译与运行（以.java 和.class结尾文件引发的考题，包括javac和java命令）

  5. 注释 单行注释 多行注释 文档注释

    ​

详细分析

一、Java语言特性：

  1. 面向对象

    两个基本概念：类、对象

    三个特性：封装、继承、多态

  2. 健壮性

    1. 吸收了C/C++语言的优点，去掉了其影响程序健壮性的部分，如指针，内存的申请和释放等，提供了相对安全的的内存管理和访问机制
    2. Java的强类型机制、异常处理、垃圾自动回收机制都是Java语言健壮性的重要保证。

  3. 跨平台特性

    通过Java语言编写的应用程序在不同的系统平台上都可以运行。”write once,Run anywhere”，”一次编译，到处运行”。

    原理：只要在需要运行 java 应用程序的操作系统上，先安装一个Java虚拟机 (JVM Java Virtual Machine) 即可。由JVM来负责Java程序在该系统中的运行。

二、Java核心机制

  1. Java虚拟机

    • JVM 是一个虚拟的计算机，具有指令集并使用不同的存储区域。负责执行指令，管理数据、内存、寄存器。
    • 对于不同的平台，有不同的虚拟机。
    • Java虚拟机机制屏蔽了底层运行平台的差别，实现了“一次编译，到处运行”

  2. 垃圾回收

    • 即对不再使用的内存空间应回收。
      在C/C++等语言中，由程序员负责回收无用内存。Java 语言消除了程序员回收无用内存空间的责任。Java提供一种系统级线程，用它来跟踪存储空间的分配情况。并在JVM空闲时，检查并释放那些可被释放的存储空间。
    • 垃圾回收在Java程序运行过程中自动进行，程序员无法精确控制和干预。
三、JDK、JRE、JVM区别与联系

  1. JDK，Java Development Kit Java 开发工具包。提供给开发人员使用，包含Java开发工具,JRE。开发工具包括：编译工具javac，打包工具jar等等。

  2. JRE，Java Runtime Environment Java 运行环境。包括JVM和Java所需的核心类库。只是运行程序则只需要JRE即可。

  3. 简单记忆：

    JDK = JRE + 开发工具集

    JRE = JVM + JavaSE标准类库

四、Java的编译与运行

用.java文件编写源码。javac命令进行编译 ，如编写一个Test.java，javac Test.java后生成一个Test.class的字节码文件。java Test运行字节码文件。
